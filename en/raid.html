  1. What's RAID ?

What's RAID? Which RAID must be done by enterprise users? We usually call RAID Redundant Array of Inexpensive Disks, its full title. It has two main purposes: one is mirroring HD data backup, or data preservation backup, another is RIAD assisted access data initiated by Stripping. The RAID1 users generally hear is this function of array mirroring backup. RAID0 is its function of acceleration. But RAID0+1 is the function of mirroring and acceleration together. The real meaning of stripping is peeling or extending, that is to combine multiple concatenation HD to one large HD. Put HD data into different HD respectively, data access would be accelerated relatively but without function of backup, any one HD breakdown would cause the whole array out of operation.

  1. Main Function of Backup RAID

More clearly, the first and foremost purpose for the server to use RAID is its function of data backup. You could think that if the content and relevant member data on a member site are lost for one night, the whole site would be almost useless. Although sometimes the disc drive of a flash drive could make backups twice, it would cost much time to back up data. And you won't remember to back up data every day. Also no one would do it carefully every day. Generally another equipment is needed to make second backup, that is tape drive. With the match of backup software, you could set files to be backed up, backup way and time to start backup without man-made operation. But the shortcoming of a tape drive is its price.

  1. Analysis of the Cause of the Most Common Breakdown of RAID

RAID operation system is unable to be started; disc architecture is partitioned or separated; RAID disk architecture declaration couldn't be found; RAID array message is lost or damaged; array card is damaged or breaks down; RAID system is unable to be started; array card breaks down; RAID rebuilds again; there is bad axle in HD; filing catalogue is disordered with messy code; conduct write operation in single HD in RAID; Array is out of order; array drops; there is bad axle in array HD; MBR is damaged; DBR is damaged; Rebuild fails or halts unexpectedly during the process; RAID breakdown strongly recommends here that don't make HD dilatation directly which is very dangerous when data is lost or other operations; don't force to cut off power suddenly; don't set HD RAID array for rebuild low-level format or deletion etc. If this RAID track is damaged and you restart system, operation system would check disk of this partition. Notice that don't let system repair as check disk of fsck etc.
